# 电商用户行为分析

> **技术栈:** MySQL · Python (Pandas, NumPy, Matplotlib) · Jupyter

## 项目概述

拿了公开的电商用户行为数据集，从浏览、收藏、加签到购买这条链路，做了数据清洗、漏斗分析和可视化，定位转化瓶颈，最后整理了一些运营方面的优化想法。

## 数据概况

| 维度 | 数值 |
|------|------|
| 行为记录 | **485,730** 条 |
| 活跃用户 | **231,912** 人 |
| 商品数 | **275,489** 个 |
| 类目数 | **5,614** 个 |
| 时间跨度 | 2017.11.26 01:00–10:00（9h） |

## 分析内容

### 1. 用户行为转化漏斗

| 环节 | 用户数 | 转化率 |
|------|-------|-------|
| 浏览 | 215,662 | — |
| 收藏 | 12,194 | 浏览→收藏 5.65% |
| 加购 | 23,143 | 浏览→加购 10.73% |
| **购买** | **10,587** | **加购→购买 45.75%** |
| **整体** | — | **浏览→购买 4.91%** |

加购→购买转化率最高（45.75%），购物车是成交关键环节。浏览→加购还有 89% 的流失，商品详情页还有优化空间。

### 2. 商品 & 类目分析

- 头部商品销量才 19 件，长尾分布很明显
- 有两个类目转化率**超过了 100%**，用户直接买，不经过加购
- 商品多的类目转化反而低，用户选不过来；小品类目标明确，决策快

### 3. 用户价值分层

| 层级 | 定义 | 占比 | 问题 |
|------|------|:----:|------|
| 低活跃 | 仅购买 1 次 | **95.09%** | 复购空间大，但没人回来 |
| 中活跃 | 购买 2-4 次 | 4.89% | 频次还能往上拉 |
| 高活跃 | 购买 ≥5 次 | 0.02% | 超级用户只有 2 个 |

95% 的用户买完一次就没影了。拉新做得再好，不如想想怎么让人回来买第二次。

## 技术实现

### 数据库 (MySQL)
- 建表导数据，48 万+ 条，加复合索引跑查询
- 用 CTE 和 RANK 窗口函数做商品排名
- 按行为类型、类目、时段、用户层级多维度聚合

### 数据清洗
- 检查缺失值和重复数据
- Unix 时间戳转 datetime
- 按 user_id + item_id + behavior_type + dt 去重

### 可视化
- Pandas + SQLAlchemy 从 MySQL 读数据
- Matplotlib 出图
